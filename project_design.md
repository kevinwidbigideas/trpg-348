# 소설 세계관 체험 시스템 기획서

> 유저가 소설 세계관에 직접 개입하여 역사의 흐름을 바꾸는 멀티 에이전트 인터랙티브 내러티브 시스템

---

## 1. 프로젝트 개요

### 핵심 콘셉트

고정된 원작 캐논(역사)과 유저의 개입으로 발생하는 분기 타임라인 사이의 긴장감을 핵심 재미 요소로 삼는다. 유저는 주인공(하준) 시점으로 세계에 진입하며, 유저의 위치와 행동 근처의 인물들이 독립적인 LLM 에이전트로 활성화되어 집단적으로 반응하고 새로운 서사를 만들어낸다.

### MVP 범위

- **1챕터 샌드박스**: 화전민 마을 + 주변 숲 범위
- **주인공 시점 단일 진입**: 하준으로 시작
- **근접 에이전트 활성화**: 유저 위치 기반으로 관련 인물만 병렬 처리
- 범위 외 지역 이동 시도 → 세계관 검증 에이전트가 자연스럽게 제한
- 1챕터 엔딩: 조이 정체 공개 (tick 6)

### 레퍼런스 시스템

| 레퍼런스 | 차용 개념 |
|---|---|
| **HAMLET (Chen et al., 2025)** | 계층적 멀티 에이전트 오케스트레이션, PAD(Perceive-Act-Decide) 모듈 |
| **StoryVerse (Wang et al., 2024)** | 추상적 이벤트 플래닝 + 세계 상태 기반 서사 생성 |
| **DiaryPlay** | Branch-and-Bottleneck 구조 — 분기는 자유롭되 주요 역사적 체크포인트로 수렴 |
| **Open-Theatre** | 계층적 메모리 시스템, 인물별 에피소딕 메모리 관리 |
| **LiveVectorLake** | 고정 RAG + 동적 RAG 이중 구조, 실시간 청크 임베딩 |
| **D&D 5e / Powered by the Apocalypse** | DC 기반 주사위 판정, 부분 성공 개념 |

## 2. IP 표준화 스키마

새로운 소설 IP를 추가할 때 동일한 구조로 넣을 수 있도록 표준화된 데이터 패키지를 정의한다.

```
IP_PACKAGE/
├── world/
│   ├── physics.json         # 마법 체계, 기술 수준, 물리 법칙
│   ├── geography.json       # 지역, 이동 수단, 이동 소요 시간
│   └── communication.json   # 통신 수단 및 조건
├── characters/
│   ├── {character_id}.json  # 인물별 페르소나 데이터
│   └── relations.json       # 관계망 그래프
├── timeline/
│   ├── canon_events.json    # 원작 역사 이벤트
│   └── causality_map.json   # 이벤트 인과관계 맵
└── index.json               # IP 메타데이터 및 버전
```

```
유저 입력
    │
    ▼
┌─────────────────────────────────────┐
│          메인 오케스트레이터           │
│  - 전체 게임 상태 관리                │
│  - 에이전트 호출 순서 결정            │
│  - RAG 쿼리 라우팅                   │
└──────────┬──────────────────────────┘
           │
    ┌──────┼──────────────────────┐
    ▼      ▼                      ▼
[시간    [세계관 검증]         [커뮤니케이션
오케스트레이터]  [난이도 산정]    게이트]
    │      │                      │
    └──────┴──────────┬───────────┘
                      ▼
           ┌──────────────────────────┐
           │   근접 인물 에이전트 풀   │ (병렬 실행)
           │   유저 위치 기반 활성화   │
           │  agent_hector            │
           │  agent_joy / agent_matt  │
           └──────────┬───────────────┘
                      ▼
              [내러티브 에이전트]
                      │
                      ▼
                  유저 출력
```

### 근접 에이전트 활성화 원칙

```
유저 위치: nameless_village
    ↓
활성화 (병렬 실행):
 ├── 헥터    (같은 location)
 ├── 조이    (같은 location)
 └── 매튜    (같은 location)

비활성화 (백그라운드 틱만 진행):
 ├── 슈네드  (다른 지역, 추격 중)
 └── 말리온  (수도)

예외 — 이벤트 연동 활성화:
 슈네드가 마을로 접근 중
 → 인접 location 진입 시점부터 활성화
```

## 4. 에이전트 상세 명세

### 4.1 시간 오케스트레이터

유저의 개입도에 따라 시간 흐름 속도를 동적으로 제어한다.

**시간 밀도 모드**

| 모드 | 조건 | 동작 |
|---|---|---|
| **Macro (스킵)** | 유저가 이벤트와 무관한 위치/행동 | 시간 요약 서술 후 다음 이벤트로 점프 |
| **Micro (확대)** | 유저가 이벤트 근처에 위치하거나 개입 시도 | 이벤트 단위로 세분화, 선택지 제공 |

```json
{
  "current_tick": 5,
  "mode": "micro",
  "active_event": "kenshin_awakening",
  "user_location": "nameless_village",
  "location_match": true,
  "time_pressure": "high"
}
```

**시간 진행 규칙**
- 유저가 핵심 이벤트에 직접 영향 → Micro 모드 유지
- 유저가 주변적 행동 반복 → 시간 조금씩 진행
- 이벤트와 완전히 무관한 행동 → Macro 스킵

### 4.2 세계관 검증 에이전트

유저 입력이 세계관 내에서 물리적으로 가능한지 판단한다.

**판정 흐름**
```
유저 입력
    ↓
RAG: 세계관 물리법칙, 마법 체계 조회
    ↓
유저 현재 상태 조회 (능력치, 보유 스킬, 위치)
    ↓
판정
 ├── 가능       → 난이도 산정 에이전트로 패스
 ├── 불가       → 차단 + 세계관 내 논리로 이유 설명 + 대안 제시
 └── 부분 가능  → 약화된 버전으로 처리 후 패스
```

**불가 응답 예시 (인터페이스 형태)**
```
❌ 유저: "기관총을 꺼냅니다"
<행동 불가>
이 세계의 기술 수준으로는 존재하지 않는 무기입니다.
가능한 행동: [석궁 발사] [마법 투사] [검술 공격]

❌ 유저: "로크빌로 이동합니다"
<이동 불가>
로크빌까지는 도보로 3일 거리입니다.
현재 상태(부상, 장비 없음)로는 무리입니다.
가능한 행동: [마을에서 준비 후 출발] [헥터에게 동행 요청]
```

### 4.3 난이도 산정 에이전트

행동의 DC(Difficulty Class)를 산정하고 RNG 판정을 실행한다.

**DC 산정 요소**

```json
{
  "action": "하운드_단독_제압",
  "actor_stats": { "combat": 20, "stealth": 14 },
  "target_stats": { "perception": 50, "defense": 70 },
  "conditions": {
    "target_wounded": false,
    "darkness": true,
    "ally_nearby": false,
    "power_gap": "target_stronger"
  },
  "base_dc": 80,
  "condition_modifiers": -10,
  "final_dc": 70
}
```

**판정 결과 단계** (d100 기준)

| 결과 | 조건 | 인터페이스 출력 |
|---|---|---|
| 대성공 | roll ≤ DC × 0.2 | `<대성공> 완벽하게 해냈습니다. 추가 이점 발생.` |
| 성공 | roll ≤ DC | `<성공> 의도한 대로 행동했습니다.` |
| 실패 | roll > DC | `<실패> 뜻대로 되지 않았습니다.` |
| 대실패 | roll ≥ 95 | `<대실패> 상황이 악화되었습니다.` |

> D&D 5e의 단일 d20 플랫 분포는 스윙이 크고, Powered by the Apocalypse의 2d6은 부분 성공 구간이 자연스럽게 생긴다. 본 시스템은 d100으로 세밀한 확률 조정이 가능하도록 설계한다.

### 4.4 커뮤니케이션 게이트 에이전트

인물 간 소통 가능 여부를 상태 기반으로 결정한다.

```
인물 A → 인물 B 소통 시도
    ↓
체크 항목:
 ├── 물리적 상태  (감금 / 폐관 / 의식불명 / 전투 중)
 ├── 수단 보유    (마법 통신 / 전령 / 직접 접촉 가능 여부)
 ├── 관계 상태    (적대 / 단절 / 신뢰)
 └── 위치 거리    (즉시 / 지연 소요 시간 산정)
    ↓
결과:
 ├── 즉시 소통 가능
 ├── 지연 소통 (전령 N일 → tick 소모)
 └── 소통 불가 (이유 명시)
```

### 4.5 인물 에이전트

각 인물은 독립적인 페르소나, 상태, 정보 범위를 가지고 동작한다.
**유저의 위치와 현재 tick 기준으로 근접한 인물만 활성화하여 병렬 실행**한다.

**에이전트 컨텍스트 구성 (매 호출 시)**

```
시스템 프롬프트:
  - 페르소나 (동기, 가치관, 말투)
  - 현재 상태 (위치, 관계, 감정)
  - 알고 있는 정보 범위 (정보 비대칭 유지)

RAG 주입:
  - 고정 RAG: 인물 페르소나, 관계, 세계관 규칙
  - 동적 RAG: 현재 분기 이벤트 청크 (tick 이하만)
  - 관계 있는 인물들의 현재 상태
  - 현재 위치의 지역 정보
```

**정보 비대칭 원칙**
- 헥터는 조이의 황녀 신분을 알지만 하준은 tick 6 이전에는 모름
- 슈네드가 접근 중이라는 사실을 마을 주민들은 모름
- 켄신은 심상공간에서만 존재 — 하준이 심상공간 진입 시에만 활성화

### 4.6 내러티브 에이전트

모든 에이전트의 결정 결과를 소설 문체로 서술하고, 판정 결과를 세계관 인터페이스 형태로 출력한다.

**출력 구조**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📖 [서술]
숲 속 공터. 어둠이 깔리기 시작한다.
갈색 머리의 소녀가 뛰쳐나오며 외쳤다.
"도망치세요!"

<퀘스트 발생 — 길 잃은 소녀와 하운드>
성공 조건: 소녀 생존 또는 하운드 처치
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
다음 행동:
[A] 소녀를 뒤에 세우고 맞선다
[B] 소녀를 데리고 도망친다
```

## 5. RAG 이중 구조

### 5.1 고정 RAG (캐논)
- 원작 세계관, 인물 페르소나, 원래 역사 타임라인
- 최초 1회 임베딩, 변경 없음
- 인덱싱 단위: 인물 / 이벤트 / 관계 / 세계관 규칙 / 지역

### 5.2 동적 RAG (분기 타임라인)
- 유저 개입 이후 발생한 사건들
- 이벤트 발생 시마다 청크 생성 + 즉시 임베딩
- LiveVectorLake 방식: 현재 세션 벡터 인덱스 + 버전 히스토리

**동적 청크 메타데이터**

```json
{
  "chunk_id": "event_0012",
  "session_id": "abc123",
  "branch_id": "branch_001",
  "tick": 3,
  "related_characters": ["hajun", "hector"],
  "location": "nameless_village",
  "canon_divergence": true,
  "content": "하준이 하운드를 단독으로 제압하여 헥터 없이 조이를 구함..."
}
```

**검색 필터링**: 현재 분기 + tick 이하 청크만 조회 → 미래 사건 오염 방지

## 6. 세션 & 분기 관리

### 6.1 저장 구조

분기 전체 청크를 저장하지 않고, **인물 상태 JSON + 분기점 요약**만 저장한다.

```json
{
  "session_id": "abc123",
  "ip_id": "dimension_348_v1",
  "created_at": "2026-05-08T12:00:00Z",
  "branches": [
    {
      "branch_id": "branch_001",
      "parent_branch": null,
      "tick": 3,
      "divergence_summary": "하운드 단독 처치 — 헥터 합류 전",
      "character_states": {
        "hajun": { "alive": true, "location": "nameless_village", "level": 2 },
        "hector": { "alive": true, "location": "nameless_village", "state": "normal" },
        "joy_josephine": { "alive": true, "location": "nameless_village", "identity_hidden": true }
      }
    },
    {
      "branch_id": "branch_002",
      "parent_branch": "branch_001",
      "tick": 5,
      "divergence_summary": "켄신 각성 전 슈네드 단독 대면",
      "character_states": {}
    }
  ]
}
```

### 6.2 분기 UI (Claude 스타일)

```
세션 시작
├── [tick 3] 하운드 단독 처치     ← 현재 플레이 중
│   └── [tick 5] 켄신 각성
│       └── [tick 6] 조이 정체 공개
└── [tick 3] 도주 선택 (다른 분기)
    └── [tick 4] 마을 이탈 후 숲에서 생존
```

- 유저가 분기점 클릭 → 해당 시점 character_states 로드 → 동적 RAG 재구성 후 재시작
- 분기점은 중요 이벤트 시 **자동 저장** (tick 기반)

## 8. 전체 이벤트 루프

```
1.  유저 입력
2.  세계관 검증 에이전트   →  불가 시 차단 + 인터페이스 형태 대안 제시
3.  시간 오케스트레이터    →  Macro/Micro 모드 결정
4.  난이도 산정 에이전트   →  DC 산정 + RNG 판정
5.  커뮤니케이션 게이트    →  관련 인물 소통 가능 여부 결정
6.  근접 인물 에이전트     →  유저 위치 기반 활성화, 병렬 실행 (RAG 주입)
7.  상태 업데이트           →  character_states 갱신
8.  동적 RAG 업데이트      →  새 이벤트 청크 임베딩
9.  분기점 여부 판단       →  중요 이벤트면 자동 저장
10. 내러티브 에이전트      →  소설 문체 서술 + 인터페이스 알림 + 선택지 생성
11. 유저 출력
```

## 10. 원작 IP 특이사항 — 348차원

### 인터페이스 시스템이 세계관에 내재
원작 소설에 게임 인터페이스(레벨/스탯/퀘스트/칭호)가 이미 세계관 내부에 존재한다.
주사위 판정 결과를 인터페이스 알림으로 자연스럽게 서술 가능하다.

```
❌ 어색한 방식:
"잠입 시도 → DC 55 / 주사위 38 → 성공"

✅ 세계관 녹여낸 방식:
<은밀한 접근 시도>
<민첩 보정 +8, 대상 수면 상태 -20>
<성공. 상대가 눈치채지 못했습니다.>
```

### 유저 포지션
유저는 지구 생존자(이세계 전이자) 포지션으로 진입.
세계관을 모르는 것이 자연스럽고, 인터페이스로 정보를 습득하는 구조가 이미 설계되어 있다.

### 주요 갈등 구도 (캐논 기반)
- **포식자 강림 저지** — 최종 목표 (tick 최후반)
- **신성제국 지구인 수배** — 중반 주요 위기 (tick 25~)
- **조이 정체 공개 및 귀환** — 1챕터 엔딩 (tick 6)
- **헥터 과거 (철사자)** — 슈네드와의 갈등 트리거

### IP 스키마 파일
`ip_schema_348.md` 참고 — 인물 JSON, 캐논 이벤트 타임라인, 세계관 검증 규칙, 1챕터 지역 범위 포함.
